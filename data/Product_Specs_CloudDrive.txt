================================================================================
TECHNICAL ARCHITECTURE
================================================================================

INFRASTRUCTURE:
- Multi-cloud deployment (AWS primary, GCP backup)
- CDN: CloudFlare Enterprise
- Database: PostgreSQL 15 (primary), MongoDB (metadata)
- Cache Layer: Redis Cluster
- Search Engine: Elasticsearch 8.x
- Message Queue: RabbitMQ

STORAGE TIERS:
1. Hot Storage (SSD): Files accessed in last 30 days
2. Warm Storage (HDD): Files accessed 31-90 days ago
3. Cold Storage (Glacier): Files accessed 90+ days ago
4. Archive (Deep Archive): Compliance retention only

AUTO-TIERING:
- Files automatically moved between tiers based on access patterns
- 40% cost savings on average
- User-configurable policies

ENCRYPTION:
- Client-side encryption: AES-256-GCM
- Key derivation: Argon2id
- Transport layer: TLS 1.3
- Zero-knowledge architecture: TechVault cannot decrypt user data

DATA CENTERS:
- Primary: US-East (Virginia), US-West (Oregon)
- Europe: Frankfurt, London, Paris
- Asia-Pacific: Tokyo, Singapore, Sydney
- Data residency: Users choose region, data never leaves

REDUNDANCY:
- 99.99% uptime SLA
- Triple redundancy within region
- Automatic failover to backup region
- RPO: 1 hour, RTO: 4 hours
